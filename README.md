## 本项目强化学习应用详解

### 整体框架

```
CoppeliaSim 仿真环境
        ↕ ZMQ通信
Python RL训练 (PPO算法)
```

---

### 核心要素

#### 1. 智能体 (Agent)
**YouBot 机器人**，需要学会自主捡球并送回回收仓。

#### 2. 环境 (Environment)
**TennisEnv 类**，封装了 CoppeliaSim 仿真场景，实现了标准 Gym 接口：
```
reset() → 开始新一局
step()  → 执行一个动作，返回观测+奖励
```

#### 3. 状态/观测空间 (State，11维)

| 维度 | 含义 | 为什么需要 |
|---|---|---|
| rx, ry | 机器人位置 | 知道自己在哪 |
| theta | 机器人朝向 | 知道自己面朝哪 |
| ball_dx, ball_dy | 最近球的方向 | 知道球在哪 |
| ball_dist | 最近球的距离 | 知道球有多远 |
| bin_dx, bin_dy | 回收仓方向 | 知道仓在哪 |
| bin_dist | 回收仓距离 | 知道仓有多远 |
| carried_ratio | 携带球数/最大携带量 | 知道自己装了多少 |
| remaining_ratio | 剩余球数/总球数 | 知道还剩多少 |

#### 4. 动作空间 (Action，3维连续)

| 维度 | 范围 | 含义 |
|---|---|---|
| vx | [-1, 1] | 前后移动速度 |
| vy | [-1, 1] | 左右平移速度 |
| omega | [-1, 1] | 旋转速度 |

YouBot 是麦克纳姆轮，支持全向移动，所以有3个独立控制维度。

#### 5. 奖励函数 (Reward)

```
每步固定         -0.02    时间惩罚（鼓励快速完成）
靠近最近球       +2×Δd   朝球移动时给正奖励
捡到一个球       +20      强化捡球行为
靠近回收仓       +1.5×Δd 携带球时引导去放球
放球入仓         +30×n    强化送球行为
全部球收集完     +150     终极目标奖励
越界             -10      惩罚撞墙
```

#### 6. 算法 (PPO)

使用 **Proximal Policy Optimization（近端策略优化）**，原因：
- 连续动作空间的主流算法
- 训练稳定，不容易崩溃
- 适合这种稀疏奖励+稠密奖励混合的任务

网络结构：
```
输入层 (11维观测)
    ↓
全连接层 256
    ↓
全连接层 256
    ↓
全连接层 128
    ↓
Actor头 → 输出动作 (vx, vy, omega)
Critic头 → 输出价值估计 V(s)
```

---

### 训练流程

```
Episode 开始
    ↓
reset(): 机器人回起点，球随机散布
    ↓
循环 (最多3000步):
    PPO策略网络 输入观测 → 输出动作
        ↓
    YouBot 执行动作 (move)
        ↓
    sim.step() 推进仿真一步
        ↓
    Lua脚本检测: 是否捡到球？是否放球？
        ↓
    计算奖励，更新观测
        ↓
    PPO收集经验 (512步后更新网络)
    ↓
Episode 结束 (全部收集完 或 超时)
    ↓
重复下一个 Episode
```

---

### 预期学习过程

| 训练阶段 | 步数 | 表现 |
|---|---|---|
| 初期 | 0~5万步 | 完全随机游走 |
| 初步收敛 | 5~15万步 | 开始朝球移动 |
| 中期 | 15~30万步 | 能捡球但不会送仓 |
| 后期 | 30~50万步 | 能捡球并送回仓 |
| 收敛 | 50万步+ | 稳定完成任务 |
